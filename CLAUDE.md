# CU-Bubbletea

Shopify storefront for **CU-Bubbletea**, a bubble tea shop based in Switzerland (DE-language market). Project owned by client **Asiainfo GmbH**.

The current scope is two tracks:
1. **Theme build** — custom Shopify theme (this repo).
2. **Data migration** — port the existing catalog and customer data from the client's old **PrestaShop** store into Shopify.

## Repo layout

This is a Shopify Online Store 2.0 theme (Horizon-style structure):

- `layout/` — `theme.liquid`, `password.liquid` (top-level page wrappers)
- `templates/` — JSON/Liquid templates per resource (`product.json`, `collection.json`, `cart.json`, …)
- `sections/` — reusable Liquid sections referenced from templates
- `blocks/` — reusable block components used inside sections
- `snippets/` — small Liquid partials
- `assets/` — JS, CSS, SVG icons. Custom shop styling lives in `assets/bubbletea-custom.css`
- `locales/` — translations. Default storefront language is `en.default.json`; **`de.json` is the customer-facing locale** since the shop ships to Switzerland (DE)
- `config/` — `settings_schema.json` (theme editor schema) and `settings_data.json` (auto-generated by Shopify admin — do not hand-edit unless necessary)

## Working on the theme

- Use the **Shopify CLI** locally: `shopify theme dev` for live preview, `shopify theme push` / `pull` to sync with the dev store.
- `settings_data.json` is overwritten by the Shopify admin. Treat changes there as outputs of the theme editor, not source of truth.
- Custom CSS belongs in `assets/bubbletea-custom.css` — keep it separate from the theme's `base.css` so future theme updates don't clobber it.

## Deployment

`deploy.sh` rsyncs the workspace to a remote VM and (optionally) brings up `docker-compose.yml`. It is **not** the Shopify deploy path — Shopify themes are pushed via `shopify theme push`. The VM script is for any companion service (e.g. a migration runner) that lives alongside the theme.

## Data migration (PrestaShop → Shopify)

Migration has **not started** yet. Source store is on PrestaShop; target is the new Shopify store. Decision still open on which approach to use:
- Shopify's official **Store Importer** app (CSV-based, free, limited fields).
- Third-party tools (Cart2Cart, Matrixify, LitExtension) — paid, more complete (customers, orders, redirects, SEO).
- Custom export from PrestaShop DB → Shopify CSV / Admin API.

When migrating, always preserve:
- Product **handles** / URLs (set up 301 redirects for old PrestaShop URLs to keep SEO).
- Customer accounts (Shopify will require password reset emails on first login regardless).
- Order history (orders are read-only after import — they cannot be re-fulfilled through Shopify).
